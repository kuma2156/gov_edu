## ✅ static을 사용하는 이유

- **클래스 자체에 소속된 변수나 메서드**를 만들기 위해 사용.
- 객체를 만들지 않아도 바로 사용 가능.

### 📌 예시
```java
class MathUtil {
    static int add(int a, int b) {
        return a + b;
    }
}

int result = MathUtil.add(3, 5); // 객체 없이 사용 가능
```
## ✔ 왜 쓸까?
공통으로 사용하는 값이나 기능을 만들 때

메모리 절약 (객체마다 복사 안 함)

---------

## ✅ 추상 메서드와 추상 클래스
- 추상 클래스: abstract 키워드를 가진 클래스
- 추상 메서드: 내용(몸통)이 없는 메서드 → 반드시 자식 클래스에서 재정의해야 함

```java
abstract class Animal {
    abstract void sound();  // 추상 메서드
}

class Dog extends Animal {
    void sound() {
        System.out.println("멍멍!");
    }
}

```

## ✔ 왜 쓸까?
- 자식 클래스가 꼭 특정 메서드를 만들게 강제하고 싶을 때
---
## ✅ 인터페이스 (interface)
- 메서드의 약속(규칙)을 정의하는 구조

- implements 키워드로 클래스가 이를 구현함

- 모든 메서드는 자동으로 public abstract

```java
interface Animal {
    void sound();
}

class Cat implements Animal {
    public void sound() {
        System.out.println("야옹~");
    }
}

```

## ✅ 참조 변수
- 객체의 주소값을 저장하는 변수
- `User user1 = new User()`; → user1은 User 객체의 주소를 참조함

## ✅ 업캐스팅
- 자식 객체를 부모 타입 변수에 담는 것
```java
Animal animal = new Dog();  // 업캐스팅
```
---------
## ✅ 싱글톤 패턴 (Singleton Pattern)
- 객체를 단 하나만 만들도록 제한하는 설계 방식
- 메모리 절약 + 전역에서 하나의 객체만 공유
```java
class Singleton {
    private static Singleton instance = new Singleton();
    private Singleton() {}  // 외부에서 new 불가

    public static Singleton getInstance() {
        return instance;
    }
}

```
