# 추상화 vs 인터페이스
- I > I extends
- c > c extends
- I > c implement
- 이외에 모든 것은 extends

i == interface , c == class

## 💡 게임비유 한 줄 정리
- > interface는 "규칙서" `직업설계서`
- > abstract class는 "공통 기능 모음집" 
- > class는 "실제 캐릭터"

## 제네릭 
- 자료형을 나중에 결정할 수 있게 해주는 문법
```java
List<String> list = new ArrayList<>();
```
- <> 안에 자료형을 넣어줌 (String, Integer 등)
- 장점: 타입 안전성 확보 + 형변환 안 해도 됨
## 컬렉션
- 여러 데이터를 묶어서 저장하고 다루는 자료구조
### 대표적인 컬렉션 종류:
- List : 순서 있고 중복 허용 (ArrayList, LinkedList)
- Set : 순서 없고 중복 없음 (HashSet)
- Map : 키-값 쌍 저장 (HashMap)
## 람다식
- 함수를 짧고 간결하게 표현하는 방식
```java
(매개변수) -> { 실행할 코드 }
```
```java
list.forEach(name -> System.out.println(name));
```
- name이란 값을 하나씩 꺼내서 System.out.println 실행
- 주로 stream, forEach, filter 등과 함께 사용

| 조건        | 설명                           |
| --------- | ---------------------------- |
| 함수형 인터페이스 | 추상 메서드 1개만 있어야 함             |
| 형태 일치     | 매개변수와 반환형이 인터페이스와 일치해야 함     |
| 문법 유연성    | 괄호/중괄호/return 생략 가능 (상황에 따라) |



## 와일드카드 
```java
public void printAll(Repogitory<?> repo) {
    repo.findAll().forEach(System.out::println);
}
```
- HighStudent, AcademyStudent 뭐든 OK!
- 하지만 repo.save(...) 같은 데이터 추가는 불가능 (타입이 뭔지 몰라서)

| 와일드카드           | 의미               | 읽기 | 쓰기 |
| --------------- | ---------------- | -- | -- |
| `<?>`           | 모든 타입 허용 (읽기 전용) | ✔️ | ❌  |
| `<? extends T>` | T의 하위 타입 (읽기 전용) | ✔️ | ❌  |
| `<? super T>`   | T의 상위 타입 (쓰기 전용) | ❌  | ✔️ |

### 🧠 쉽게 기억하기
- extends → "읽기 전용" (get만 가능)
- super → "쓰기 전용" (save만 가능)
- ? → "뭐든지 가능하지만 안전하게 하려면 읽기만"

## 익명클래스
- 보통 한 번만 쓰는 클래스를 간단하게 만들 때 사용
```java
button.setOnClickListener(new OnClickListener() {
    @Override
    public void onClick(View v) {
        System.out.println("버튼 클릭!");
    }
});
```
| 구분        | 설명               |
| --------- | ---------------- |
| 이름        | 없음               |
| 용도        | 일회성 클래스 빠르게 만들 때 |
| 특징        | 코드 짧게, 간단한 구현 가능 |
| 자주 사용되는 곳 | 이벤트, 콜백, 스레드 등   |
