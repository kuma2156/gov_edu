# 컬렉션

✅ 2. 자료 구조 제공
컬렉션 프레임워크는 다양한 자료 구조를 제공합니다. 상황에 따라 적절한 구조를 선택할 수 있어요.

List → 순서 있는 데이터 저장 (ex: ArrayList, LinkedList)

Set → 중복 없는 데이터 저장 (ex: HashSet, TreeSet)

Map → 키-값 쌍으로 저장 (ex: HashMap, TreeMap)

| 구분       | 중복 허용      | 순서 유지      | 주요 구조      | 예시            |
| -------- | ---------- | ---------- | ---------- | ------------- |
| **List** | O (중복 허용)  | O (인덱스 있음) | 배열, 연결 리스트 | 이름 목록, 순차 처리  |
| **Set**  | ❌ (중복 제거)  | 경우에 따라 다름  | 집합         | 태그, 유일한 값 모음  |
| **Map**  | 키: ❌, 값: O | 경우에 따라 다름  | 키-값 쌍      | 전화번호부, 설정값 저장 |
